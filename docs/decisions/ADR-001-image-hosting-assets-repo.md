# 📋 ADR-001: Image Hosting in Separate Assets Repository

## 📌 Status

**Accepted** — 2026-04-13

## 🤔 Context

HomeRacker uses images in README files, GitHub Pages, and MakerWorld descriptions. Storing these directly in the source repos creates problems:

- **History bloat**: Every image change (optimization, re-render, replacement) permanently grows the git history, even after files are deleted from the working tree
- **Clone size**: Contributors clone the full history including all past image versions
- **Dual licensing**: `homeracker` (CC BY-SA 4.0) and `homeracker-exclusive` (source-available) need different licenses for their images

### Alternatives Considered

| Approach | Verdict | Reason |
|---|---|---|
| Keep images in repo | ❌ Rejected | History accumulates with every image change |
| Git LFS | ❌ Rejected | Bandwidth risk from forks/clones on public repo |
| Orphan branch | ❌ Rejected | Too much workflow friction (worktrees, context switching) |
| Google Drive / CDN | ❌ Rejected | External dependency, link rot risk |
| GitHub issue uploads | ❌ Rejected | No version control, opaque URLs |

## 🔧 Decision

Host all manually-created images in a dedicated **[kellerlabs/assets](https://github.com/kellerlabs/assets)** repository.

### Structure

```
assets/
├── homeracker/           # CC BY-SA 4.0
│   ├── LICENSE
│   ├── img/              # README + GitHub Pages images
│   └── models/           # MakerWorld description images
└── homeracker-exclusive/ # Source-available license
    ├── LICENSE
    └── models/           # MakerWorld description images
```

### What Goes Where

| Image Type | Location | Tracking |
|---|---|---|
| Manual images (photos, diagrams, logos) | `kellerlabs/assets` repo | Tracked in assets |
| Auto-generated render PNGs (`export-png.sh`) | Source repo `*/renders/*.png` | Tracked in source |
| MakerWorld render previews (`render-mw-modules.sh`) | Source repo `*/renders/*.png` | Tracked in source |

### URL Pattern

```
https://raw.githubusercontent.com/kellerlabs/assets/main/<repo>/<path>
```

### Gitignore Strategy

Source repos block all image extensions and whitelist only auto-generated renders:

```gitignore
# Images — hosted in kellerlabs/assets repo
*.png
*.jpg
*.jpeg
*.gif
*.webp
*.svg
*.ico
*.xcf

# Whitelist: auto-generated render previews + favicon
!**/renders/*.png
!favicon.ico
```

## 📊 Consequences

- **Positive**: Clean git history in source repos, proper dual-licensing, contributors get smaller clones
- **Positive**: Auto-gen PNGs stay coupled to source code (regenerated by scripts, whitelisted in gitignore)
- **Negative**: Extra repo to maintain, images referenced via absolute URLs instead of relative paths
- **Negative**: `md-to-mw.py` base64 embedding only applies to local relative images; assets-hosted images are passed through as URLs
